# الدليل الشامل لأنواع البيانات في StarUML (مشروع إدارة الثغرات)

هذا الجدول يوضح الحقول وأنواعها الدقيقة كما هي مطبقة في قاعدة البيانات، ليتم إدخالها في المخطط الهندسي.

### 1. جدول الحسابات (profiles)
| الحقل | النوع في StarUML | الوصف |
| :--- | :--- | :--- |
| `id` | `uuid` | المفتاح الأساسي |
| `display_name` | `text` | اسم المستخدم |
| `avatar_url` | `text` | رابط الصورة الشخصية |
| `role` | `text` | الصلاحيات (admin/user) |

### 2. جدول نتائج الفحص (scan_results)
| الحقل | النوع في StarUML | الوصف |
| :--- | :--- | :--- |
| `id` | `uuid` | المفتاح الأساسي |
| `name` | `text` | اسم الفحص (مثلاً: فحص سيرفر الإنتاج) |
| `target` | `text` | العنوان المستهدف (IP or URL) |
| `tool` | `text` | الأداة (NMAP, NIKTO, SQLMAP, FFUF) |
| `status` | `text` | الحالة (completed, running, pending) |
| `critical_count` | `integer` | عدد الثغرات الحرجة المكتشفة |
| `total_findings` | `integer` | المجموع الكلي للنتائج |
| `user_id` | `uuid` | المعرف البرمجي لصاحب الفحص |

### 3. جدول الثغرات المكتشفة (vulnerabilities)
| الحقل | النوع في StarUML | الوصف |
| :--- | :--- | :--- |
| `id` | `uuid` | معرف الثغرة الداخلي |
| `cve_id` | `text` | الرقم العالمي للثغرة (مثل CVE-2024-...) |
| `cvss_severity` | `text` | مستوى الخطورة (Critical, High, Medium) |
| `cvss_score` | `float` | الدرجة الرقمية (0.0 - 10.0) |
| `status` | `text` | حالة المعالجة (Open, Resolved) |

### 4. جدول الأصول (scanned_assets) - (View)
| الحقل | النوع في StarUML | الوصف |
| :--- | :--- | :--- |
| `id` | `uuid` | معرف الجهاز المكتشف |
| `ip_address` | `text` | عنوان IP الخاص بالجهاز |
| `hostname` | `text` | اسم النطاق أو اسم الجهاز |
| `os` | `text` | نظام التشغيل (إذا تم اكتشافه) |
| `open_ports` | `text` | المنافذ المفتوحة (مثلاً: 80, 443) |
| `risk` | `text` | تقييم الخطر الكلي للجهاز |

---

## ملاحظة هامة حول العلاقات:
جدول **scanned_assets** هو في الواقع **واجهة عرض (View)** تستخلص بياناتها من **scan_results**.
- **برمجياً:** لا يحتاج لمفتاح أجنبي لأنه "اشتقاقي".
- **هندسياً (في StarUML):** يجب رسم علاقة "Dependency" (سهم متقطع) من `scan_results` إلى `scanned_assets` لتوضيح أن بيانات الأصول تعتمد على وجود نتائج فحص سابقة.
